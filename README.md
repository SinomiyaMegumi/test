项目启动书：轻量级电商平台管理系统 
项目名称   轻量级电商平台管理系统 
版本号   v3.0 (Spring Boot 3.x)

文档日期   2026-03-23

架构风格   MVC / RESTful API

核心语言   Java 17

项目背景与概述
本项目旨在构建一个高内聚、低耦合的轻量级电商平台后端系统。基于最新的 Spring Boot 3.x 生态，结合 MyBatis 持久层框架，为前端（Web/小程序/App）提供稳定、高效的数据接口服务。系统覆盖了从用户注册、商品浏览、购物车管理到订单结算的全流程电商业务场景，同时集成了后台管理与数据统计功能，适用于中小型电商项目的快速落地与二次开发。

技术栈选型

2.1 核心框架
运行环境: JDK 17+
核心框架: Spring Boot 3.2.10
ORM框架: MyBatis (配合 MySQL 8.0)
构建工具: Maven

2.2 安全与认证
认证机制: JWT (JSON Web Token) 无状态认证
权限控制: 自定义拦截器 + 角色权限校验
密码加密: BCrypt 或其他安全哈希算法

2.3 数据存储与文件
关系型数据库: MySQL 8.0
文件存储: 本地文件系统 (支持大文件上传，最大限制 100MB)
图表数据: 集成 ECharts 数据结构支持

2.4 其他关键组件
API 规范: RESTful 风格
跨域处理: CORS 全局配置
异常处理: 全局统一异常捕获机制 (@ControllerAdvice)
分页插件: MyBatis 分页插件或 PageHelper

核心功能模块详解

本系统划分为以下十大核心业务模块，各模块间通过 Service 层解耦：

3.1  用户中心模块
功能描述: 负责平台用户的生命周期管理。
关键特性:
  用户注册、登录（账号密码）、密码找回。
  个人信息修改（头像、昵称、手机号）。
  JWT 令牌签发与刷新，实现无状态会话管理。
  登录状态拦截与权限校验。

3.2  商品管理模块
功能描述: 平台商品的核心数据维护。
关键特性:
  商品信息的增删改查 (CRUD)。
  多级商品分类管理。
  商品浏览历史记录追踪。
  商品收藏/取消收藏功能。

3.3  店铺管理模块
功能描述: 服务于入驻商家的店铺运营。
关键特性:
  商家店铺信息创建与维护。
  店铺关注/收藏功能。
  店铺资质与公告管理。

3.4  购物车模块
功能描述: 用户购物前的临时存储与预处理。
关键特性:
  添加商品至购物车（校验库存与状态）。
  购物车内商品数量修改、选中/取消选中。
  批量下单前的大数据计算（总价、总数量）。

3.5  订单管理模块
功能描述: 交易核心流程，连接支付与物流。
关键特性:
  订单创建（生成唯一订单号）。
  订单状态流转（待支付->待发货->待收货->已完成/已取消）。
  订单详情查询与列表分页。
  订单评价系统（评分与文字评论）。

3.6  地址管理模块
功能描述: 物流配送的基础数据支撑。
关键特性:
  收货地址的增删改查。
  默认地址标记与自动选取逻辑。
  地址簿容量限制管理。

3.7  文件资源模块
功能描述: 系统多媒体资源的统一入口。
关键特性:
  图片、文档上传接口。
  大文件上传支持 (配置上限 100MB)。
  文件访问路径映射与资源清理。

3.8  广告与运营模块
功能描述: 首页内容展示与营销推广。
关键特性:
  轮播图 (Banner) 配置与管理。
  广告位投放管理。
  首页推荐数据聚合接口。

3.9  统计分析模块
功能描述: 为管理层提供数据决策支持。
关键特性:
  销售数据报表（日/月/年维度）。
  用户增长统计。
  ECharts 图表数据源接口封装。

3.10  后台管理模块
功能描述: 系统管理员的超级控制台。
关键特性:
  管理员账号与角色权限分配。
  系统基础配置动态调整。
  全平台数据监控看板。

系统架构设计

4.1 目录结构规范
项目遵循标准的 Maven 分层架构：
src/main/java/com/project/platform
├── controller/    # 控制层：接收请求，参数校验，返回视图对象(VO)
├── service/       # 业务层：核心逻辑处理，事务控制
├── mapper/        # 持久层：SQL 映射接口
├── entity/        # 实体层：数据库表映射对象 (DO)
├── dto/           # 传输层：前端传入的数据对象
├── vo/            # 视图层：返回给前端的数据对象
├── config/        # 配置层：CORS, Swagger, MyBatis, JWT 等配置
├── utils/         # 工具类：Result 统一返回，JwtUtil, FileUtil 等
└── exception/     # 异常处理：自定义异常与全局处理器

4.2 交互流程
Request: 前端发起 HTTP 请求 -> Controller
Auth: 拦截器校验 JWT Token
Process: Service 层执行业务逻辑 (事务控制)
Data: Mapper 层与 MySQL 交互
Response: 封装为统一 Result 格式返回前端

开发环境与启动指南

5.1 前置要求
安装 JDK 17 并配置 JAVA_HOME
安装 Maven 3.8+
安装 MySQL 8.0 并创建数据库 (字符集推荐 utf8mb4)
IDE 推荐：IntelliJ IDEA 2023+

5.2 配置步骤
数据库初始化:
   在 MySQL 中执行项目提供的 .sql 脚本，创建表结构及初始数据。
配置文件修改:
   打开 src/main/resources/application.yaml
   修改数据库连接信息：
          spring:
       datasource:
         url: jdbc:mysql://localhost:3306/your_db_name?useSSL=false&serverTimezone=Asia/Shanghai
         username: root
         password: your_password
     
   配置 JWT 密钥 (jwt.secret) 及过期时间。
   配置文件上传路径 (file.upload-path)。

5.3 启动命令
在项目根目录下执行：
mvn clean install
mvn spring-boot:run

或直接在 IDE 中运行 PlatformApplication.java。

5.4 验证成功
启动后访问：
健康检查: http://localhost:8080/api/health (示例)
接口文档: 若集成 Swagger/Knife4j，访问 http://localhost:8080/doc.html
